Introduction to HTML

HyperText Markup Language — the skeleton of every webpage. Not a programming language, it's a markup language.

What is HTML?

HTML stands for HyperText Markup Language. It defines the structure and content of web pages using a system of elements/tags. Browsers parse HTML and render it visually.

  • HyperText — text with links to other documents
  • Markup — annotating text with tags to give it meaning
  • Language — a standardized system of rules

HTML vs CSS vs JavaScript

LayerRoleAnalogy
HTMLStructure & ContentSkeleton of a building
CSSStyling & LayoutPaint, décor, layout
JavaScriptBehavior & InteractivityElectricity & plumbing

HTML Versions Timeline

VersionYearKey Feature
HTML 1.01993Basic hypertext
HTML 2.01995Forms, tables
HTML 3.21997Scripting, applets
HTML 4.011999CSS separation, accessibility
XHTML 1.02000XML-based strict HTML
HTML52014Semantic tags, APIs, multimedia — current standard

How Browsers Work (Brief)

When you open a webpage, the browser:

  1. Fetches HTML from server (HTTP/HTTPS)
  2. Parses HTML → builds DOM (Document Object Model)
  3. Fetches CSS → builds CSSOM
  4. Combines DOM + CSSOM → Render Tree
  5. Layout → Paint → Composite → Display
💡
Key InsightHTML is parsed top-to-bottom. Scripts in <head> block rendering unless async/defer is used.

A Minimal HTML Page

html
<!-- This tells the browser: "I'm HTML5" -->
<!DOCTYPE html>
<html lang="en">

<head>
  <!-- Metadata goes here, NOT visible to user -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Page</title>
</head>

<body>
  <!-- Visible content goes here -->
  <h1>Hello, World!</h1>
  <p>This is a paragraph.</p>
</body>

</html>

Document Structure

The anatomy of an HTML document — every valid HTML page has this structure.

DOCTYPE Declaration

DOCTYPE is not an HTML tag — it's an instruction to the browser about which HTML version is being used.

html
<!-- HTML5 (simplest) -->
<!DOCTYPE html>

<!-- Old HTML4 (complex, rarely used now) -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
  "http://www.w3.org/TR/html4/loose.dtd">
⚠️
Quirks ModeIf DOCTYPE is missing, browsers switch to "Quirks Mode" — they simulate old buggy behaviour. Always include DOCTYPE!

The <html> Element

Root element — wraps everything. The lang attribute is important for SEO and accessibility.

html
<html lang="en">   <!-- English -->
<html lang="hi">   <!-- Hindi -->
<html lang="en-US"> <!-- US English -->

The <head> Section

Contains metadata — information about the document, not displayed directly to users.

html
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">

  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Page title (shown in browser tab) -->
  <title>Page Title</title>

  <!-- Link external CSS -->
  <link rel="stylesheet" href="style.css">

  <!-- Favicon -->
  <link rel="icon" href="favicon.ico">

  <!-- Internal CSS (not recommended for large projects) -->
  <style>
    body { font-family: sans-serif; }
  </style>

  <!-- SEO meta tags -->
  <meta name="description" content="Page description for SEO">
  <meta name="keywords" content="html, notes, web">
</head>

The <body> Section

Contains all visible content — text, images, links, forms, etc. Everything users see and interact with.

Attributes provide additional information about elements. They appear in the opening tag.

data-* Attributes (Important!)

Store custom data in HTML that JavaScript can read via dataset. Very common in modern frontend.

html + js
<!-- HTML -->
<button data-user-id="42" data-action="delete">Delete</button>
<div data-theme="dark" data-max-items="10"></div>

// JavaScript
const btn = document.querySelector('button');
console.log(btn.dataset.userId);   // "42"
console.log(btn.dataset.action);   // "delete"
// Note: data-user-id → dataset.userId (camelCase)

Boolean Attributes

These attributes are either present (true) or absent (false). Their value doesn't matter.

html

<input required autofocus>           <!-- multiple booleans -->
<video autoplay muted loop></video>   <!-- video booleans -->
<details open></details>             <!-- visible by default -->

HTML provides tags for semantic text formatting. Understand semantic vs presentational distinction.

Semantic Text Tags

html

<em>Emphasized text</em>        <!-- Italic + semantic emphasis -->
<mark>Highlighted text</mark>   <!-- Yellow highlight -->
<del>Deleted text</del>         <!-- Strikethrough -->
<ins>Inserted text</ins>        <!-- Underline (insertion) -->
<sub>subscript</sub>           <!-- H₂O -->
<sup>superscript</sup>         <!-- x² -->
<code>inline code</code>       <!-- monospace code -->
<kbd>Ctrl+C</kbd>              <!-- keyboard input -->
<abbr title="HyperText Markup">HTML</abbr>  <!-- abbreviation -->
<cite>Book Title</cite>        <!-- citation -->
<q>Short inline quote</q>     <!-- adds quotation marks -->
<time datetime="2024-01-15">January 15</time>

Presentational vs Semantic (Interview Topic!)

❌ Presentational (Old Way)
<b>Bold text</b>
<i>Italic text</i>
<u>Underline</u>
<s>Strikethrough</s>
✅ Semantic (Modern Way)
<strong>Important</strong>
<em>Emphasized</em>
<ins>Inserted</ins>
<del>Deleted</del>
💡
Interview Gold<strong> conveys importance semantically (screen readers say "important"); <b> just makes text bold visually. Same visual output, different meaning. Always prefer semantic tags.

Links & Anchors

The <a> tag is the foundation of the web — it creates hyperlinks.

The <a> Element

html
<!-- External link -->
<a href="https://google.com">Google</a>

<!-- Open in new tab -->
<a href="https://google.com" target="_blank" rel="noopener noreferrer">
  Google (new tab)
</a>

<!-- Relative link (same site) -->
<a href="/about">About Page</a>
<a href="../images/photo.jpg">Photo</a>

<!-- Anchor (same page scroll) -->
<a href="#section2">Jump to Section 2</a>
<h2 id="section2">Section 2</h2>

<!-- Email link -->
<a href="mailto:ash@email.com?subject=Hi&body=Hello">Email Me</a>

<!-- Phone link -->
<a href="tel:+919999999999">Call Me</a>

<!-- Download link -->
<a href="/resume.pdf" download="Ash_Resume.pdf">Download Resume</a>

<!-- Link with image -->
<a href="https://example.com">
  <img src="logo.png" alt="Company Logo">
</a>
🔒
Security: rel="noopener noreferrer"Always add this when using target="_blank". Without it, the opened page can access your window via window.opener — a security vulnerability called "reverse tabnapping".

Images & Media

Embedding images, responsive pictures, iframes, and more.

The <img> Element

html

Responsive Images — <picture>

html

<img
  src="small.jpg"

<!-- picture element: art direction (different image for screen size) -->
<picture>
  <source media="(max-width: 600px)" srcset="mobile.jpg">
  <source media="(max-width: 1200px)" srcset="tablet.jpg">
  <img src="desktop.jpg" alt="Hero image">  <!-- fallback -->
</picture>

<!-- WebP with fallback -->
<picture>
  <source type="image/webp" srcset="image.webp">

</picture>

HTML5 provides built-in form validation without JavaScript.

Validation Attributes

html

Semantic HTML5

Semantic tags give meaning to content — crucial for SEO, accessibility, and clean code.

Why Semantic HTML?

  • SEO — Search engines understand page structure better
  • Accessibility — Screen readers navigate by semantic landmarks
  • Maintainability — Code is self-documenting
  • Developer Experience — Easier to understand at a glance

Page Layout Semantic Tags

html
<body>

  <!-- Site-wide header (logo, nav) -->
  <header>
    <h1>My Blog</h1>
    <nav>
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
      </ul>
    </nav>
  </header>

  <!-- Main content area (only one per page!) -->
  <main>

    <!-- Self-contained piece (blog post, news article) -->
    <article>
      <header>
        <h2>Post Title</h2>
        <time datetime="2024-03-18">March 18, 2024</time>
      </header>
      <p>Article content...</p>
      <footer>Written by Ash</footer>
    </article>

    <!-- Part of page (related to main, removable = aside) -->
    <aside>
      <h3>Related Posts</h3>
    </aside>

    <!-- Thematic grouping of content -->
    <section id="skills">
      <h2>My Skills</h2>
    </section>

  </main>

  <!-- Site-wide footer -->
  <footer>
    <p>&copy; 2024 Ash. All rights reserved.</p>
  </footer>

</body>

ElementWhen to useTest
<article>Self-contained, redistributable content"Would this make sense in an RSS feed?"
<section>Thematic group of content with a heading"Does it have its own heading?"
<div>Generic container for styling/JS"No semantic meaning needed?"

Canvas & SVG

Two ways to draw graphics in HTML5 — each with different use cases.

Canvas Element

Pixel-based drawing using JavaScript. Good for animations, games, image processing.

html + js
<canvas id="myCanvas" width="400" height="200">
  Canvas not supported  <!-- fallback text -->
</canvas>

// JavaScript drawing
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Rectangle
ctx.fillStyle = '#00d9a6';
ctx.fillRect(10, 10, 150, 100);

// Circle
ctx.beginPath();
ctx.arc(200, 100, 50, 0, Math.PI * 2);
ctx.fillStyle = '#7c6fff';
ctx.fill();

// Text
ctx.font = '20px Arial';
ctx.fillStyle = 'white';
ctx.fillText('Hello Canvas!', 10, 180);

SVG (Scalable Vector Graphics)

Vector-based, scales infinitely without quality loss. Good for icons, logos, charts.

html
<!-- Inline SVG -->
<svg width="200" height="100" xmlns="http://www.w3.org/2000/svg">
  <circle cx="50" cy="50" r="40" fill="#00d9a6"/>
  <rect x="120" y="10" width="70" height="80" fill="#7c6fff"/>
  <text x="10" y="95" font-size="12" fill="white">Hello SVG</text>
</svg>

Canvas vs SVG

FeatureCanvasSVG
TypeRaster (pixel-based)Vector (coordinate-based)
ScalingBlurs on zoomCrisp at any size
DOMNo DOM nodes per shapeEach shape is a DOM element
EventsNeeds manual hit detectionNative event listeners per element
PerformanceBetter for many objects/animationsBetter for few complex shapes
Use caseGames, image filters, chartsIcons, logos, interactive diagrams

Script Loading Strategies

html
<!-- Regular: blocks HTML parsing until script downloads + runs -->
<script src="app.js"></script>

<!-- async: download in parallel, run ASAP (may run before DOM ready) -->
<script src="analytics.js" async></script>

<!-- defer: download in parallel, run AFTER HTML parsed (maintains order) -->
<script src="app.js" defer></script>

<!-- For modules (ES6) -->
<script type="module" src="main.js"></script>
💡
Best PracticeUse defer for scripts that need the DOM. Use async for independent scripts like analytics. Place <script defer> in <head> — it's better than putting it at end of body.

Resource Hints

  <link rel="preload" href="font.woff2" as="font" crossorigin>

🔥 Interview Questions

Most-asked HTML interview questions with detailed answers. Click each to expand.

Fundamentals

Q1 What is the difference between HTML and HTML5? ▶
HTML5 additions:
  • Semantic elements: <header>, <footer>, <article>, <section>, <nav>, <aside>, <main>
  • New form input types: date, range, color, email, tel, number, search
  • Audio/Video: <audio>, <video> without plugins
  • Canvas API for 2D graphics
  • Web Storage: localStorage, sessionStorage
  • Web Workers, WebSocket, Geolocation API
  • Removed deprecated tags: <font>, <center>, <strike>
  • Simpler DOCTYPE: <!DOCTYPE html>
Q2 What is the DOM? How does it relate to HTML? ▶
DOM = Document Object Model
When a browser parses HTML, it creates a tree-like object representation called the DOM. This is NOT the HTML itself — it's a live in-memory representation.
  • HTML is the source text; DOM is what the browser creates from it
  • JavaScript interacts with the DOM, not the HTML file directly
  • DOM can change (via JS) without changing the original HTML file
  • document.getElementById('id') traverses the DOM tree
Q3 What's the difference between <b> and <strong>, <i> and <em>? ▶
Both pairs produce identical visual output, but differ in meaning:
  • <b> — purely visual bold, no semantic meaning
  • <strong> — semantic importance; screen readers emphasize it
  • <i> — visual italic (used for technical terms, foreign words)
  • <em> — semantic stress emphasis; screen readers change tone
Rule: Prefer <strong> and <em> for semantic content.
Q4 What are void elements / self-closing tags? ▶
Void elements cannot have any children (no content) and don't require a closing tag.
  • <br>, <hr>, <img>, <input>
  • <meta>, <link>, <base>
  • <area>, <col>, <embed>, <param>, <source>, <track>, <wbr>
In HTML5, writing <br/> is allowed but the slash is optional and ignored.
Q5 What is the difference between id and class attributes? ▶
  • id: Unique per page — only ONE element can have a given id. Used for JS targeting (getElementById) and anchor links (#section). Higher CSS specificity.
  • class: Multiple elements can share a class. One element can have multiple classes (class="btn primary large"). Used primarily for CSS styling.
Interview answer: id = unique identifier (like a name), class = category/group.
Q6 What is the purpose of the alt attribute in img tags? ▶
  • Accessibility: Screen readers read alt text aloud for visually impaired users
  • Fallback: Shown if image fails to load
  • SEO: Search engines index alt text for image search
  • Empty alt (alt=""): For decorative images — screen readers skip them
Missing alt is both an accessibility violation and bad for SEO.

Semantic & Structure

Q7 What is semantic HTML? Why does it matter? ▶
Semantic HTML uses tags that describe the meaning of their content, not just appearance.
  • SEO: Search engines understand page structure (headings, nav, main content)
  • Accessibility: Screen readers use landmarks (<nav>, <main>) for navigation
  • Developer Experience: Code is self-documenting and easier to maintain
  • Code Quality: Less div soup
Example: <nav> vs <div class="nav"> — both look the same, but <nav> tells the browser "this is navigation".
Q8 What's the difference between <article>, <section>, and <div>? ▶
  • <article>: Self-contained, independently distributable content (blog post, news article, product card). Test: "Can this stand alone with full meaning?"
  • <section>: Thematic grouping of content, always with a heading. Part of a larger whole.
  • <div>: Generic block container with no semantic meaning. Use for layout/styling when no semantic element fits.
<article> can contain <section>s. <section>s can contain <article>s.
Q9 What is the <meta charset="UTF-8"> tag? Why is it important? ▶
It declares the character encoding of the HTML document. UTF-8 can represent every character from every language.
  • Without it, browsers may misinterpret characters, showing "mojibake" (garbled text)
  • Must be the first element in <head>, before <title> — because <title> might contain special chars
  • UTF-8 is the universal standard — use it always
Q10 What is the viewport meta tag? What does initial-scale=1.0 mean? ▶
Without the viewport meta, mobile browsers zoom out to show the full desktop layout, making text tiny.
  • width=device-width: Set viewport width to device screen width
  • initial-scale=1.0: Don't zoom in/out by default (1:1 scale)
  • Essential for responsive design with CSS media queries to work correctly
  • Without it, @media (max-width: 768px) may not trigger on mobile

Forms & Links

Q11 What is the difference between GET and POST in forms? ▶
  • GET: Data appended to URL as query string (?name=ash&age=21). Visible in browser history. Limited to ~2000 chars. Use for search, filters — idempotent operations.
  • POST: Data in request body. Not visible in URL. No size limit (practically). Use for login, registration, file upload — state-changing operations.
Security note: GET data is logged in server access logs and browser history. Never use GET for passwords.
Q12 Why do we use rel="noopener noreferrer" with target="_blank"? ▶
Security concern: When a page opens a new tab with target="_blank", the new page can access the original via window.opener.
  • A malicious site could use window.opener.location = 'phishing.com' to redirect your original tab — this is called "reverse tabnapping"
  • rel="noopener": Sets window.opener to null in the new tab
  • rel="noreferrer": Also prevents sending the Referer header (privacy) + implies noopener
  • Modern browsers set noopener by default for target="_blank", but still include it for older browser support
Q13 What is the difference between <input type="button">, <input type="submit"> and <button>? ▶
  • <input type="button">: Doesn't submit form. Needs JS onclick handler.
  • <input type="submit">: Submits the form. Text from value attribute only.
  • <button type="submit">: Same as above but can contain HTML (icons, styled text). Default type inside a form is "submit".
  • <button type="button">: Doesn't submit. Preferred over <input type="button">.
  • <button type="reset">: Clears form fields.
Best practice: Always specify type on <button> — default is "submit" which can cause accidental form submissions.

Advanced Topics

Q14 What is the difference between async and defer on <script> tags? ▶
  • Normal <script>: Stops HTML parsing, downloads script, executes it, then continues parsing.
  • async: Downloads in parallel with HTML parsing. Executes as soon as downloaded (pauses parsing). Order not guaranteed. Good for: analytics, ads (independent scripts).
  • defer: Downloads in parallel. Executes AFTER HTML is fully parsed. Order guaranteed. Good for: scripts that manipulate the DOM.
Rule of thumb: Use defer for your app scripts, async for third-party independent scripts.
Q15 What is the difference between localStorage, sessionStorage, and cookies? ▶
  • localStorage: 5-10MB, persists until manually cleared, NOT sent to server, domain-scoped
  • sessionStorage: ~5MB, cleared when tab closes, NOT sent to server, tab-scoped
  • Cookies: ~4KB, configurable expiry, SENT with every HTTP request, can be HttpOnly (no JS access)
Security: Cookies with HttpOnly flag prevent XSS theft. Use cookies for auth tokens, localStorage for preferences.
Q16 What is ARIA and when should you use it? ▶
ARIA = Accessible Rich Internet Applications. A set of attributes that make web content more accessible to people with disabilities, especially for screen readers.
  • Use when native HTML semantics are insufficient (custom widgets, dynamic content)
  • First rule: Don't use ARIA if native HTML does the job (<button> is already accessible)
  • Key attributes: role, aria-label, aria-hidden, aria-live, aria-expanded
Q17 What is the difference between Canvas and SVG? ▶
  • Canvas: Pixel-based (raster). Drawn with JS API. No DOM per shape → better performance for many objects. Blurs on zoom. Good for: games, image processing, real-time visualization.
  • SVG: Vector-based. XML-based → each shape is a DOM element. Native events per element. Scales infinitely. Good for: icons, logos, maps, interactive diagrams.
Q18 What are data-* attributes? How do you use them? ▶
Custom attributes to store private data in HTML elements, accessible via JavaScript's dataset property.
  • Naming: data- prefix + any name: data-user-id, data-theme
  • JS access: element.dataset.userId (hyphen-case → camelCase)
  • CSS access: [data-theme="dark"] { ... }
  • Use case: Pass server-side data to JS without hidden inputs or global variables
Q19 What are Web Workers? What problem do they solve? ▶
JavaScript runs on a single thread. Heavy computations block the UI, making the page freeze.
  • Web Workers run scripts in background threads, off the main thread
  • Communication via postMessage / onmessage
  • Cannot access DOM directly (no document, window)
  • Types: Dedicated Workers (one page), Shared Workers (multiple pages), Service Workers (proxy/offline)
Q20 What is the difference between <script> in head vs end of body? ▶
  • In <head> without defer/async: Blocks HTML parsing — page shows blank until script loads. Bad for performance.
  • End of <body>: HTML parsed first, DOM ready, then script executes. Historically the workaround for the blocking issue.
  • In <head> with defer: Modern best practice. Downloads in parallel, executes after DOM ready. Same result as end of body but cleaner.
Today's answer: Use <script defer> in <head>.
